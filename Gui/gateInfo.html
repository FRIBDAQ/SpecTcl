Information available from gates

SpecTcl gates are used to impose conditions that restrict when a spectrum is incremented. Gamma gates can be further used to create a spectrum that is folded with respect to one or more decay energies. The browser line displaying each gate provides the gate type code in the "Type" column. Additional gate type dependent information is available by drilling down into individual gates. Gate specific information is divided into the following categories:

  • Slice gates
  • 2-d gates (contours and bands)
  • 1-d Gamma gates
  • 2-d Gamma gates (gamma contours and gamma bands)
  • Bitmask gates (And mask, equal mask, not mask)
  • Compound gates (and, or and not gates)

Note that constant gates (true and false), don't have any additional information and therefore cannot be drilled into

Slice gates

Slice gates display their gate type code "s" in the Type column. Drilling down provides the following information:

  • The parameter the gate is defined on
  • The values of the parameter that define the lower and upper limits of the slice

2-d gates (contours and bands)

2-d gates provide their gate type code on the same line as the object in the Type column. Contour gates are type "c" while band gates are of type "b". These gates type both provide the following information when drilled into.

  • The name of the X parameter on which the gate is defined (named as "X : parameter name"
  • The name of the Y parameter on which the gate is defined (named as "Y : parameter name"
  • A folder named Points. When opened, this folder provides the x/y coordinates of the points that describe the gate boundaries.

1-d gamma gates

1-d gamma gates provide their gate type on the same line as the object in the Type column. The type code of a gamma slice is "gs". Drilling into a gamma slice gate provides the following:

  • A folder named "Parameters" which, when opened, lists the parameters on which the gate was defined.
  • The lower and upper limits of the gate.

2-d gamma gates

A 2d gamma gate can either be a gamma contour or a gamma slice. 2d gamma gates display their gate type code (gc for gamma contour, gs for gamma slices) on the same line as the gate. Drilling into the gate provides:

  • A folder named "Parameters" that, when opened, lists the parameters on which the gate was set.
  • A folder named "Points" that, when opened, lists the points that define the gate shape.

Bitmask gates

Bitmask gates provide their type code on the same line as the gate. There are three types of bitmask gates:

  • am - and mask gates, A parameter is bitwise anded with a mask. If the result is equal to the mask, the gate is satisfied.
  • em - equal mask gates, If the parameter is bitwise the same as the mask, the gate is satisfied.
  • nm - not mask gates, The parameter is anded with the bitwise comlement of the mask. If the result is equal to the mask, the gate is satisfied.

Drilling into a bitmask gate provides:

  • The name of the parameter on which the gate was set.
  • The value of the bitmask

Compound gates

A compound gate is a gate that is dependent on other gates. Each compound gate lists its gate type in the Type column on the same line as the gate in the browser (* for and gates, + for or gates, and - for not gates). Drilling into the gate provides the list of gates that the gates depends on.

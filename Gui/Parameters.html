Parameter context menus and selection actions

If there are unfamiliar terms in this page, you should consult the Glossary help page

Parameter objects are stored in the Paramters top level folder. The top level folder has an associated context menu as does each parameter. These context menus are triggered when you right click over the folder or parameter. In addition, parameters have a selection action that is triggered when a parameter is double-clicked.

The Parameter top level folder context menu

The parameter top leve folder context menu is activated by right clicking the Parameters top level folder. This menu has the following commands:

  • Pseudo... Allows you to create a scripted pseudo parameter. Scripted pseudo parameters functions that are computed by applying a Tcl script to a subset of the parameters in each event.
  • Refresh TreeRefreshes the browser tree. Normally the tree will refresh itself as needed whenever changes are made. However if changes are made the GUI is not aware of, you may need to manually refresh the tree.

The parameter object context menu

The parameter object context menu is activated by right clicking over a parameter object. The parameter context menu has the following commands:

  • Edit... Edits the parameter definition. If the parameter is not a tree parameter, it will be turned into a tree parameter by this action. The parameter low, high and bins suggestions must be set, while the parameter units may be set.
  • Pseudo...Allows the parameter to be replaced by a pseudo parameter or, if the parameter is already a pseudo, allows you to edit its definition.

The parameter selection action.

The parameter selection action is the same as the Parameter context menu's Edit.. entry. The parameter definition is edited. If the parameter is not yet a treeparameter, one will be made for it by this operation.

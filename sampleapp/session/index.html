Session Demonstrations

These pages are examples of the use of sessions to control access and contain persistent state.

Each time the following pages are loaded, the Session_Match method is called to find the session associated with the given session identifier.

The session identifier 'new' is special, and causes a new session to be created.

Session identifier 'kill' (where id is a valid session id) will destroy the nominated session, and create a new one.

Building a tailored SpecTcl.

The SpecTcl skeleton comes with a template Makefile. There are only two things you'll usually need to know and adjust.

  1. The INSTDIR Makefile variable at the top of the Makefile points to the installation directory of SpecTcl. In many cases this can just be modified when upgrading SpecTcl to get the build to work on a newer verson of the software. If you are moving SpecTcl to some other institution where SpecTcl is installed differently this is the only variable you'll need to change to get allow the Makefile to build for the same version in this new environment.
  2. The OBJECTS variable is the set of .o's that you want to build. If you have added any sourc files to the build (likely) you need to add them to this.

Once you have made the appropriate changes you can: 

    make

To build SpecTcl.

Modifying Skeleton Software.

Once you have copied the skeleton software, you must build your own customized version of SpecTcl. The files which you have copied from the SpecTcl Skel directory include a Makefile. If you have added files (in addition to UserCode.cpp), or changed names, modify this Makefile to reflect these changes.

In most cases, tailoring SpecTcl for a data set means:

  1. Writing one or more event processors that convert raw events into parameters.
  2. Setting up the event analysis pipeline so that the event processors are run in the appropriate order.
  3. Building the tailored SpecTcl.

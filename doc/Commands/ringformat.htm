ringformat command

SpecTcl Home  General Information User Guide Programmer's Guide Obtaining and Installing

Syntax Synopsis Description Examples

Syntax

    ringformat major?.minor?
    
    Where:
       major is the major version of the data format.
       minor is the mionr version of the data format.

Synopsis

Specifies the format in which the NSCLDAQ data are in when using a ring data source.

Detailed Description

Ring data sources (-format ring) cover a variety of formats. From nscldaq-11.0 and on, each run begins with a ring format item that describes the format of datas in the ring. If, however SpecTcl joins an online run in progress, it won't see this item until the next run begins. Furthermore, nscldaq-10.x runs have no ring format items.

The ringformat command specifies for SpecTcl how to treat data from ring formatted data sources. If SpecTcl encounters a ring format data item, the format specification in that item will override that specified in the ringformat command.

The format is specified as a decimal number with a major and minor version field. If the minor version is omitted, a value of 0 is used by default. See the Examples section below for some examples of ring format version specifications.

Examples 

    
    ringformat 11.0;      # [1]
    ringformat 10.1;      # [2]
    ringformat 10;        # [3]
  1. Sets the format of ring items to that of NSCLDAQ-11.0
  2. Sets the format of ring items to that of NSCLDAQ-10.1 note that all NSCLDAQ-10.x versions use the same format of ring item.
  3. Sets the format of ring items to that of NSCLDAQ-10.0. This form illustrates that if the minor version is not supplied, it defaults to 0
